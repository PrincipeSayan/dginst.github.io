OpenTimestamps: a Deep Down Tutorial

A timestamp proves that some data existed prior to a given point in time. The OpenTimestamps protocol defines an open standard consisting in a set of operations for creating provable blockchain timestamps and later independently verifying them.
While anyone could realize a timestamp with the permissionless blockchain(s) by paying the appropriate transaction fees, there are public OpenTimestamps calendar servers, free to use without any registration or API key.
This webpage is an interface to create timestamp proofs using the OpenTimestamps public calendar servers (e.g. btc-ots.dgi.io): Bitcoin's blockchain is used as a timestamp notary, i.e. the proof is attested in a Bitcoin block header.
These proofs can be verified independently from any OpenTimestamps server or facility, using a local Bitcoin Core node; anyway, since a web-page cannot access the local filesystem, this page relies on public block-explorers for verification.

Calculating the Data File Hash Value

Calculate the hash value associated to the data file to be timestamped.
No internet connectivity is required: the hash is calculated inside your browser without disclosing your document to third parties, preserving your privacy.




Submitting for Attestation

Submit the hash value to multiple OpenTimestamps calendar servers for attestation of its existence (timestamp) and save the OTS submission locally; internet connectivity is required.
However, the receipt is not a proof yet, i.e. it cannot be verified immediately: it is incomplete as it takes a few hours for the timestamp to get confirmed by the Bitcoin blockchain.
To avoid the inefficiency of one blockchain transaction for every timestamp, a calendar server provides aggregation of multiple documents (in a Merkle tree data structure) and performs their attestation in a single transaction (actually attesting the tree root).
In time, the hash value will be attested in a Bitcoin block header and the receipt will become upgradable to attestation proof.



Loading OTS Receipt/Proof From File

Load an existing OTS receipt/proof from a local file; no internet connectivity required.



Displaying OTS Receipt/Proof File Info

Display the info included in the OTS receipt/proof, including the actual commitment operations and attestations (if any); no internet connectivity is required.



Upgrading the OTS Receipt/Proof

Attempt the upgrade of the OTS receipt/proof with attestations that might be available from the OpenTimestamps calendars; internet connectivity is required.
To be upgraded to proof status, incomplete receipts require the assistance of the remote calendars used for timestamping: each calendar has to provide its attestation as path to a Bitcoin block header. Upgrading a OTS receipt adds the path to the Bitcoin block header, making it a self-contained calendar-independent OTS proof.
Even proofs can be updated, if there is a calendar that can add its own attestation to the ones previously provided by other calendars.



Verifying OTS Proof

Verify the attestations included in the OTS proof; internet connectivity is required.
OTS proofs can be verified independently from any OpenTimestamps server or facility, using a local Bitcoin Core node; anyway, since a web-page cannot access the local filesystem, this page relies on public block-explorers for verification.
Proof upgrading is first attempted contacting the OpenTimestamps calendars before verification: if an upgrade is available, then it is downloaded and the upgrade is used for verification.
