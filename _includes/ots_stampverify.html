Timestamp and Verify

Use the box below to:

  1. create an attestation proof (timestamp):
    when a file is dropped in the box, its hash value is calculated locally inside the browser without disclosing the document to third parties, preserving privacy.
    The hash is then submitted to multiple redundant public OpenTimestamps servers (e.g. btc.ots.dgi.io) to obtain an attestation proof of its existence (timestamp).
    A submission receipt is immediately downloaded (as .ots filetype); this is not an attestation proof yet: it is incomplete and it cannot be verified immediately, as it can take up to few hours for the transaction to be included in the blockchain.
    In time, the hash value will be attested in a block: then, the servers it has been submitted to will be able to upgrade its submission receipt to attestation proof. As long as at least one server is properly functioning, the timestamp will be ensured.
  2. upgrade a submission receipt or update/verify an attestation proof (*.ots):
    if the file dropped in the box is a submission receipt (attestation proof), first an upgrade (updated) proof is downloaded from the calendar servers if available; then the proof, if any, is verified using public Bitcoin block-explorers (e.g. blockstream.info).

Drop here a file to timestamp it or a receipt/proof (*.ots) to verify it.
